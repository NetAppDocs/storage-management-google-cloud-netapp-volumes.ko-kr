= Google Cloud NetApp Volumes 역할 할당
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
Google Cloud NetApp Volumes 작업을 시작하려면 NetApp Console 에서 멤버에게 올바른 역할을 할당해야 합니다. 자세한 내용은 다음을 참조하세요.link:https://docs.netapp.com/us-en/console-setup-admin/reference-iam-predefined-roles.html["모든 서비스에 대한 NetApp Console 액세스 역할에 대해 알아보세요."^] .



== 서비스 계정 멤버에게 역할 할당

. 로그인하세요 https://docs.netapp.com/us-en/console-setup-admin/task-logging-in.html["NetApp Console"^] .
. *관리*를 선택한 다음 *ID 및 액세스*를 선택합니다.
. 작업 메뉴를 선택하세요image:screenshot_gallery_options.gif["세 개의 점이 있는 아이콘입니다."] 역할을 할당하려는 구성원 옆에 있는 *역할 추가*를 선택합니다.
. 역할을 추가하려면 대화 상자의 단계를 완료하세요.
+
.. 폴더 또는 프로젝트 선택: 멤버에게 권한이 부여되어야 하는 리소스 계층 수준을 선택합니다.
+
폴더를 선택하면 해당 멤버는 해당 폴더 내에 있는 모든 항목에 대한 권한을 갖게 됩니다.

.. *범주* 드롭다운에서 *애플리케이션* 역할 범주를 선택합니다. https://docs.netapp.com/us-en/console-setup-admin/reference-iam-predefined-roles.html["액세스 역할에 대해 알아보세요"^].
.. *역할* 드롭다운에서, 선택한 폴더나 프로젝트와 연관된 리소스에 대한 권한을 멤버에게 제공하는 역할을 선택합니다.
+
* Important:* 다음 역할 중 하나 또는 둘 다를 추가해야 합니다.

+
*** _Google Cloud NetApp Volumes 관리자_ 또는
*** _Google Cloud NetApp 볼륨 뷰어_


.. 역할 추가: 조직 내 추가 폴더나 프로젝트에 대한 액세스 권한을 제공하려면 *역할 추가*를 선택하고 다른 폴더나 프로젝트 또는 역할 범주를 지정한 다음 역할 범주와 해당 역할을 선택합니다.


. *새로운 역할 추가*를 선택하세요.
